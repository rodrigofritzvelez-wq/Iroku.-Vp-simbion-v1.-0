Overview (Technical & Functional)

Simbyon is a lightweight, offline-capable stack of nano-apps that together form a verification-first symbiotic network. Each app targets a core function: governance, traceability, mesh communications, energy & verification.

  • Design principle: Nano-apps ≤ 15KB each, zero external runtime deps (vanilla HTML/CSS/JS).
  • Core module: AMI (Autonomous Merit Intelligence) — coherence verifier and policy guard.
  • Deployment model: PWA/sandbox for demos; edge nodes (phones/drones/nodes) for mesh; SimbioChain for immutable anchoring.
Intended for demos, pilots and GitHub Pages. All simulated metrics are clearly labelled for demo purposes.

Core apps (v1.0)

  • Symbiolytics — AMI engine, score + certificate generator (demo: 12.5KB).
  • SymbioChain Explorer — ledger explorer & node dashboard (demo: ~9.1KB).
  • SymbioChat — collaborative dialogue + virtue wheel (seeded IA personas).
  • SymbioMesh — offline mesh comms visualiser & node share.
  • SolarMesh — energy management, tokenization & self-heal simulation.

AMI — role & responsibilities

AMI is the coherence gatekeeper. It evaluates proposals and evidence on five axes: Origin, Lineage, Semantics, Integrity, Risk. It also acts as an ethical guard (detects attempts to produce fraudulent receipts, illicit content, or manipulative behaviour) and issues immutable certificates (demo tokens: SYM-XXXX).

// score example (demo)
score = round(origin*0.25 + lineage*0.20 + semantics*0.20 + integrity*0.15 + risk*0.20)

Deployment & dev notes

  • Local demo: open HTML file in browser (works offline).
  • PWA: add manifest + service worker for offline caching (recommended for field demos).
  • Production anchors: IPFS / SimbioChain anchors and signatures for certificates.
  • Edge nodes: phones, dedicated Sotolín-inspired devices, later drone/nanosat swarms.
